# WI-006 - Public Project Polish and CI

Status: proposed

## Goal

Improve the public GitHub project surface so it is easier for humans and agents to understand, validate, and trust.

## Why

This repo is now public and should serve as both project memory and portfolio evidence. Public polish increases credibility and makes the project easier to resume.

## Scope

In scope:

- GitHub Actions validation workflow
- badges in README
- repo topics and description
- issue templates
- contribution notes
- public milestone notes
- release notes for v0.1.0-public-lab

Out of scope:

- changing project scope
- phone actions
- large binary artifacts
- proprietary firmware publishing

## Deliverables

- CI workflow running governance validation and unit tests
- README badge
- issue templates
- first release notes or CHANGELOG entry

## Acceptance

- GitHub Actions passes on main
- README links to validation behavior
- public repo clearly says what is and is not claimed
- no large or private artifacts added

## Safety

No device interaction.
